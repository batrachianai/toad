## Tier 1: Gold Standard Sources

These sources conduct rigorous research. Cite with high confidence.

### Nielsen Norman Group (nngroup.com)
- **Focus**: Usability research, interaction design, information architecture
- **Quality**: 30+ years of research, eyetracking studies, user testing
- **Use for**: Heuristics, best practices, interaction patterns
- **Example citation**: "Users spend 80% of time above the fold (NN/g, 2018)"

### Baymard Institute (baymard.com)
- **Focus**: E-commerce UX, checkout optimization
- **Quality**: 19+ years of checkout research, 151+ guidelines
- **Use for**: Checkout flows, cart design, form optimization
- **Example citation**: "48% abandon due to extra costs like shipping (Baymard, 2024)"

### Gov.uk Design System
- **Focus**: Accessible, user-tested government services
- **Quality**: Rigorous user research, accessibility-first
- **Use for**: Form design, accessibility patterns, plain language
- **Example citation**: "Address lookup reduces errors by 20% (GDS research)"

## Tier 3: Platform Guidelines

Official platform guidance. Essential for platform-specific design.

### Apple Human Interface Guidelines
- **URL**: developer.apple.com/design/human-interface-guidelines
- **Use for**: iOS/macOS design patterns, accessibility
- **Key sections**: Components, inputs, navigation, accessibility

### Material Design (Google)
- **URL**: m3.material.io
- **Use for**: Android design, component specifications
- **Key sections**: Components, foundations, guidelines

### Microsoft Fluent Design
- **URL**: fluent2.microsoft.design
- **Use for**: Windows/Microsoft products
- **Key sections**: Components, patterns, accessibility

### Web Standards
- **WCAG** (w3.org/WAI/WCAG21/quickref) - Accessibility guidelines
- **ARIA** (w3.org/WAI/ARIA/apg) - Accessible widget patterns
- **Web.dev** (web.dev) - Web performance and best practices

## How to Cite in Recommendations

### Strong Citation (high confidence)
Use when citing Tier 1 sources with specific data:
> "Research shows 35% of users abandon checkout if forced to create an account (Baymard Institute, 2024). Recommend offering guest checkout."

### Supporting Citation (moderate confidence)
Use when citing Tier 2 sources or combining sources:
> "Industry benchmarks suggest onboarding flows over 5 steps see significant drop-off (Appcues, Mixpanel reports). Consider reducing to 3 steps."

### Pattern Reference (general guidance)
Use when citing platform guidelines or best practices:
> "Following iOS Human Interface Guidelines, touch targets should be minimum 44×44 points for comfortable tapping."

### When No Citation Available
Be transparent about source:
> "Based on general UX principles, users expect the 'Submit' button at the bottom of forms. The current placement at the top may cause confusion."

## Data to Always Verify

Some statistics are frequently cited but may be outdated or misattributed. Always verify:

| Commonly Cited | Reality Check |
|----------------|---------------|
| "Users decide in 0.05 seconds" | Based on 2006 study; still directionally true but context matters |
| "Users don't scroll" | Outdated; users do scroll but above-fold still matters |
| "F-pattern reading" | Depends on content type; not universal |
| "3-click rule" | No research supports this; task completion matters more than clicks |
| "Mobile users are always rushed" | Oversimplification; context varies widely |

When citing statistics, prefer:
1. Recent data (within 2-3 years)
2. Specific source with methodology
3. Relevant context (industry, platform, user type)
